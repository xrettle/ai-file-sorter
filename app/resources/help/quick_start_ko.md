# 빠른 시작 가이드

AI File Sorter는 사용자의 검토와 승인을 거친 뒤 파일을 정리할 수 있도록 도와줍니다.

AI는 분석을 주도하고 카테고리, 하위 카테고리, 이름을 제안합니다. AI가 파일을 직접 변경하지는 않습니다. 이동이나 이름 변경은 검토한 변경 사항을 사용자가 확인한 뒤 앱이 수행합니다.

## 1. 폴더 선택

정리할 폴더를 선택하려면 **Browse**를 사용하세요.

대표적인 예:

- `Downloads`
- 정리할 데스크톱 폴더
- 외장 드라이브의 폴더
- 프로젝트 보관 폴더

## 2. 앱이 수행할 작업 선택

기본 옵션을 사용해 앱이 다음 작업을 수행할지 결정하세요.

- 파일을 카테고리 폴더로 분류
- 이미지 분석
- 문서 분석
- 지원되는 파일에 대해 이름 변경 제안 제공

이름 변경 제안만 원한다면 해당 이름 변경 전용 모드를 활성화하세요.

## 3. 분류 스타일 선택

목적에 가장 잘 맞는 스타일을 선택하세요.

- **Default**: 일반적인 사용
- **More categories**: 더 세밀한 그룹화
- **More consistent**: 비슷한 파일 사이의 더 강한 일관성

앱이 더 제한된 카테고리 이름만 사용하도록 하려면 카테고리 화이트리스트도 활성화할 수 있습니다.

## 4. 분석 시작

**Analyze and categorize files**를 클릭하세요.

앱은 선택한 폴더를 스캔하고 필요한 정보를 수집한 뒤 검토 목록을 준비합니다.

## 5. 적용 전에 검토

검토 창에서는 다음 내용을 확인할 수 있습니다.

- 제안된 카테고리
- 선택 가능한 하위 카테고리
- 지원되는 파일에 대한 이름 변경 제안
- 최종 대상 경로

무엇이든 확정하기 전에 제안을 수정하거나 거부할 수 있습니다.

## 6. 변경 사항 적용

확인하면 앱이 필요한 폴더를 만들고 이동 또는 이름 변경을 수행합니다.

## 7. 마지막 실행 취소

변경 사항을 적용한 뒤 되돌리고 싶다면 메뉴에서 **Undo last run**을 사용하세요.

실행 취소는 가장 최근에 확인한 정리 실행을 대상으로 합니다. 앱에 기록된 실행 기록을 사용해 가능한 경우 파일을 이전 위치로 되돌리고 지원되는 이름 변경을 취소합니다.

가장 좋은 결과를 위해 같은 폴더에서 다른 큰 정리를 시작하기 전에 실행 취소를 사용하세요.

## 8. 검토 결과에서 배우기

검토 창에서 카테고리를 승인하면 앱은 그 로컬 결정을 기억하고 다음 실행에서 힌트로 사용할 수 있습니다. 이 과정은 AI 모델을 훈련하거나 변경하지 않습니다.

학습된 예시는 별도의 로컬 데이터베이스에 저장되므로 일반 카테고리 캐시를 지워도 삭제되지 않습니다. 이 로컬 학습 데이터를 삭제하려면 **Settings -> Reset learned behavior**를 사용하세요.

## 알아두면 좋은 점

- 앱은 같은 파일을 다시 처리하지 않고 일관성을 높이기 위해 로컬 캐시를 사용합니다.
- 앱은 검토 단계를 먼저 보여주지 않고 자동으로 변경 사항을 적용하지 않습니다.
- 더 세밀한 제어가 필요하면 이미지 옵션과 문서 옵션을 각각 펼칠 수 있습니다.

## 문제가 있어 보인다면

먼저 다음 사항을 확인하세요.

- 선택한 폴더가 정말 원하는 폴더인지
- 관련 분석 옵션이 활성화되어 있는지
- 이름 변경 전용 모드가 결과를 원하지 않는 방식으로 제한하고 있지 않은지
- 카테고리 화이트리스트가 제안을 너무 좁히고 있지 않은지

추가 도움이 필요하면 **Help -> FAQ**를 여세요.
